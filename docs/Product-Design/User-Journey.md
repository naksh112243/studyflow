# User Journey

## Purpose

This document describes how a government exam aspirant uses the app throughout a normal day.

The goal of the app is simple:

> Remove planning friction so the student only focuses on studying.

The app should never make the user think about planning.
It should simply tell them what to study next.

---

# Target User

- Government exam aspirant
- Studies full-time
- Fixed daily routine
- Wants consistency
- Does not want to plan every day
- Uses the app as a study timetable, not as a task manager

---

# One-Time Setup

The user installs the app.

During setup, the user enters:

- Wake-up time
- Sleep time
- Daily study timetable
    - Subject
    - Start time

The app automatically calculates session durations.

After setup, the user should rarely need to edit the timetable.

---

# Daily Journey

## 1. Wake Up

The user wakes up.

Normally nothing needs to be done.

If the user wakes up later than planned,
they can open the menu and tap:

Just Woke Up

The app automatically adjusts today's schedule.

The user never manually rearranges sessions.

---

## 2. Open App

The home screen immediately shows:

Current Subject

Example:

Now

Mathematics

07:00 – 08:30

No dashboard.

No statistics.

No distractions.

The user instantly knows what to study.

---

## 3. Study Session

The user studies the current subject.

The app quietly tracks time.

No unnecessary notifications.

No motivational popups.

No animations.

The app stays calm.

---

## 4. Finish Session

The user taps:

✓ Complete

Immediately:

- Session becomes completed.
- Next session becomes active.
- Progress updates.

Nothing else happens.

---

## 5. Break

Breaks are automatically inserted by the app.

The user never creates break sessions manually.

After the break,
the next study session becomes active.

---

## 6. Missed Session

If the user cannot study a session,
they have two choices.

Option 1

Delete Session For Today

The session disappears only for today.

The app automatically adjusts the remaining schedule.

Option 2

Simply leave it incomplete.

The app continues with the remaining schedule.

The entire day should never fail because of one missed session.

---

## 7. Late Wake-up

If the user wakes up late,

they tap:

Just Woke Up

The app automatically:

- shifts sessions
- adjusts timings
- preserves the study routine

No manual editing is required.

---

## 8. Continue Studying

Throughout the day,
the user repeats only one action:

Complete Session

The app handles everything else.

---

## 9. End of Day

The final session is completed.

The app shows:

Today's Progress

Example

5 / 6 Sessions Completed

No long reports.

No analytics.

No graphs.

The day ends.

---

# Daily User Actions

The user should only perform these actions:

- Complete Session
- Just Woke Up (when needed)
- Delete Session For Today (when needed)

Everything else should happen automatically.

---

# Design Philosophy

The user should never think:

"What should I study now?"

The app should always answer that question immediately.

---

# Success Criteria

A successful day means:

- The user opened the app.
- The app instantly showed the current subject.
- The user studied.
- The user tapped Complete.
- The app managed the rest.

The fewer decisions the user makes,
the better the product becomes.
