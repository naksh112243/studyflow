# UI Design Rules

## Design Philosophy

The interface should feel calm, clean, and effortless.

The design should disappear into the background so the user can focus entirely on studying.

Every visual element must have a purpose.

If something does not improve usability, it should not exist.

---

# Design Inspiration

Inspired by the simplicity of:

* Apple
* Linear
* Notion
* Arc Browser
* Raycast

Not by colorful productivity apps.

---

# Overall Feel

The application should feel:

* Calm
* Minimal
* Premium
* Modern
* Lightweight
* Distraction Free

The interface should never feel busy.

---

# Color Palette

Use only neutral colors.

Primary

Black

White

Gray Shades

Accent

One single accent color for interactive elements.

The accent color should be subtle and never dominate the interface.

Avoid multiple bright colors.

---

# Theme

Light Mode

White background

Soft gray cards

Dark text

Dark Mode

Near-black background

Dark gray cards

White text

Avoid pure black (#000000) and pure white (#FFFFFF) for large surfaces.

Use softer shades for better readability.

---

# Typography

Large headings.

Comfortable body text.

High readability.

Never use decorative fonts.

Maintain generous line spacing.

Information should be readable at a glance.

---

# White Space

White space is part of the design.

Never try to fill empty areas.

A clean layout is better than a crowded one.

---

# Layout

Use a simple vertical layout.

Current Session

↓

Next Session

↓

Nothing Else

The user should never need to search for important information.

---

# Cards

Every study session should be displayed as a clean card.

Rounded corners.

Soft shadows.

Comfortable padding.

No heavy borders.

---

# Buttons

Buttons should feel obvious.

Primary Action

Complete

Secondary Actions

Hidden inside the menu.

Avoid placing multiple buttons on the screen.

---

# Icons

Use a single icon style throughout the app.

Prefer simple outline icons.

Icons should support text, not replace it.

---

# Animations

Animations should feel natural.

Fast.

Smooth.

Subtle.

Examples

Card updates

Progress changes

Bottom sheet opening

Avoid flashy animations.

Avoid unnecessary motion.

---

# Transitions

Every transition should help orientation.

Nothing should suddenly appear or disappear.

Keep interactions smooth.

---

# Home Screen

The current study session must receive the most attention.

Everything else should remain visually secondary.

The Home screen should always feel calm.

---

# Focus

One screen.

One purpose.

One current task.

Everything else should stay in the background.

---

# Consistency

Spacing

Typography

Corner radius

Icons

Buttons

Animations

All should remain consistent across the application.

Consistency creates trust.

---

# Accessibility

Readable font sizes.

Good contrast.

Large touch targets.

Easy to use with one hand.

Never rely only on color to communicate information.

---

# Empty States

Empty screens should never feel broken.

Instead, guide the user.

Example

"No timetable found."

"Create your daily routine to get started."

---

# Loading States

Keep loading minimal.

Whenever possible,

display existing content immediately.

Avoid blocking the interface.

---

# Error Messages

Error messages should be calm and helpful.

Never blame the user.

Instead of

"Error!"

Prefer

"Something went wrong. Please try again."

---

# Success Messages

Keep feedback subtle.

Avoid celebrations.

Avoid confetti.

Avoid achievement popups.

A small confirmation is enough.

Example

"Session completed."

---

# Things We Will Never Add

* Flashy gradients
* Neon colors
* Excessive animations
* Gamification
* Floating widgets
* Popups
* Cluttered dashboards
* Advertisement banners
* Unnecessary statistics

---

# Final Design Goal

The interface should feel invisible.

The user should forget they are using an app.

They should simply open it, study, complete the session, and continue their day.

The design succeeds when nothing distracts the user from learning.

Motion Philosophy

Ye Apple se inspired hai.

Nothing should move unless it helps the user understand something.

Matlab:

Session complete hua? Card smoothly upar slide ho.
Next session active hua? Soft fade.
Hamburger khula? Gentle slide.

❌ Lekin card bounce nahi karega.

❌ Zoom nahi karega.

❌ Confetti nahi udegi.

❌ Fireworks nahi honge. 😂
