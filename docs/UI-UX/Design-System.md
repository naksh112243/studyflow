# Design System

## Purpose

This document defines the visual language of the application.

Every screen, component, interaction, and animation should follow these rules.

The objective is to create a calm, premium, distraction-free experience that helps users focus on studying.

---

# Core Philosophy

The interface should disappear.

The user should focus on studying, not on learning how to use the application.

Every visual element must have a purpose.

If an element does not improve usability, remove it.

---

# Product Personality

The application should feel:

* Calm
* Minimal
* Premium
* Modern
* Timeless
* Consistent
* Reliable
* Distraction Free

The interface should never feel busy or overwhelming.

---

# Visual Style

Inspired by the design principles of:

* Apple
* Linear
* Notion
* Arc Browser
* Raycast

We adopt their philosophy of simplicity, consistency, and attention to detail rather than copying their appearance.

---

# Color System

The application should use a neutral color palette.

Primary Colors

* White
* Black
* Gray Shades

Accent

* One accent color only.

The accent color exists to communicate interaction.

It should never be used as decoration.

Rules

* Never use multiple accent colors.
* Never rely on color alone to communicate information.
* Every color should have a purpose.

---

# Theme

Support both:

* Light Mode
* Dark Mode

Both themes should feel identical except for brightness.

Avoid pure white and pure black for large surfaces.

Prefer softer shades for better readability and reduced eye fatigue.

---

# Typography

Typography should prioritize readability above everything else.

Rules

* Large headings
* Comfortable body text
* Clear hierarchy
* Generous line spacing
* No decorative fonts

The user should understand every screen within one second.

---

# Spacing System

Spacing is a design element.

Empty space improves focus.

Rules

* Use generous spacing.
* Keep layouts breathable.
* Never crowd information.
* Maintain consistent spacing throughout the application.

---

# Layout System

The application follows a simple vertical hierarchy.

Current Session

↓

Complete Button

↓

Next Session

↓

Everything Else

The user should never search for important information.

---

# Design Hierarchy

Every screen must follow the same visual priority.

1. Current Session
2. Complete Button
3. Next Session
4. Remaining Time
5. Secondary Information

This hierarchy should never change.

---

# Cards

Cards are the primary content container.

Rules

* Rounded corners
* Comfortable padding
* Soft elevation
* Clean separation
* No heavy borders

Cards should feel light and effortless.

---

# Corner Radius

Use one consistent corner radius across the application.

Do not mix multiple corner styles.

Consistency creates familiarity.

---

# Buttons

Buttons communicate importance through placement rather than color.

Primary Button

* Complete

Secondary Actions

* Hidden inside the Hamburger Menu

Avoid multiple competing primary buttons.

---

# Icons

Use one icon family throughout the application.

Icons should be:

* Simple
* Outline Style
* Easy to recognize
* Consistent

Icons should support text.

Never replace text with icons alone.

---

# Navigation

Navigation should remain minimal.

Primary Navigation

* Home

Secondary Navigation

* Bottom Sheets
* Dialogs

Avoid deep navigation.

Avoid unnecessary pages.

---

# Motion System

Motion should explain.

Never entertain.

Animations should be:

* Fast
* Smooth
* Subtle

Motion should help users understand changes in state.

Nothing should move unless it improves understanding.

---

# Shadows

Use soft shadows only when necessary.

Avoid floating effects.

Depth should feel natural and unobtrusive.

---

# Borders

Prefer spacing over borders.

Use dividers only when they improve readability.

Avoid visual noise.

---

# Feedback

Success

Small and quiet confirmation.

Error

Clear and calm explanation.

Loading

Minimal loading indicators.

Never interrupt the user's focus unnecessarily.

---

# Empty States

Empty screens should never feel broken.

Always explain what the user should do next.

Example

"No timetable found."

"Create your daily timetable to get started."

---

# Accessibility

Accessibility is a requirement.

Rules

* Large touch targets
* Good contrast
* Readable typography
* One-hand usability
* Color should never be the only indicator

Everyone should be able to use the application comfortably.

---

# Responsive Design

The experience should remain consistent across:

* Mobile
* Tablet
* Desktop
* Installed PWA

The layout may adapt, but the interaction should remain familiar.

---

# Consistency Rules

Every screen should reuse the same:

* Color palette
* Typography
* Spacing
* Corner radius
* Cards
* Buttons
* Icons
* Motion
* Shadows

Consistency is more important than creativity.

---

# Things We Will Never Do

* Multiple accent colors
* Rainbow color palettes
* Flashy gradients
* Glassmorphism
* Neumorphism
* Excessive blur
* Heavy shadows
* Decorative illustrations
* Confetti animations
* Achievement popups
* Visual clutter

The interface should remain timeless.

---

# Design Principles

Less UI.

More Focus.

Less Decoration.

More Clarity.

Less Interaction.

More Studying.

---

# Definition of Great Design

The user should never stop to admire the interface.

The best compliment is:

"I forgot I was using the app."

The interface becomes invisible.

The study becomes the focus.

That is successful design.


# UI Golden Rules

Every new feature must satisfy these questions:

1. Can this be done with fewer taps?

2. Does this reduce planning effort?

3. Does this help the user study?

4. Does this make the interface simpler?

5. If this feature is removed, does the app become worse?

If the answer is "No",

the feature should not be added.
