# Moodboard

## Purpose

This document defines the emotional direction of the product.

It answers one question:

**"How should the application feel?"**

This document guides every future UI decision.

---

# Product Personality

The application should feel like a calm study companion.

It should never feel like a productivity machine.

It should quietly stay in the background while helping the user remain consistent.

---

# Emotional Goals

The user should feel:

* Calm
* Focused
* Organized
* In Control
* Comfortable
* Confident

The interface should never create stress or urgency.

---

# Core Keywords

Minimal

Calm

Premium

Clean

Modern

Timeless

Focused

Reliable

Quiet

Intentional

---

# Inspiration

The product takes inspiration from the design philosophy of:

* Apple
* Linear
* Notion
* Arc Browser
* Raycast

We borrow principles, not visual copies.

---

# Visual Direction

Large white space.

Simple layouts.

Minimal color usage.

Clear typography.

Rounded corners.

Soft shadows.

No visual clutter.

The design should breathe.

---

# Color Emotion

White

Clean

Black

Focus

Gray

Balance

Accent Color

Attention

Every color should have a purpose.

---

# Motion

Motion should explain.

Never entertain.

Animations should help users understand state changes.

They should never distract from studying.

---

# Typography

Readable.

Comfortable.

Elegant.

Never decorative.

Information should always be readable within a second.

---

# Interaction

One tap whenever possible.

Minimal navigation.

No unnecessary confirmations.

No hidden complexity.

---

# User Experience

The user should never think:

"How does this app work?"

Instead they should immediately understand what to do.

---

# Things We Want Users To Say

"This feels peaceful."

"I don't have to think."

"It just works."

"I can focus on studying."

"I open it, study, and close it."

---

# Things We Never Want Users To Say

"There are too many options."

"I don't know where to tap."

"This feels complicated."

"This is distracting."

"I need a tutorial."

---

# Design Principles

Remove everything that does not help the user study.

Every screen should answer one question.

Every interaction should reduce effort.

Every animation should have a reason.

Every pixel should earn its place.

---

# Final Vision

The application should feel invisible.

The user should not admire the interface.

They should simply forget it exists and continue studying.

That is the highest level of good design.
